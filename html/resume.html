Compilers and Interpreters

Course Project | Spring 2020 Prof. Preeti Ranjan Panda

MIPS Architecture and Design

Microprocessor without interlocked pipelined stages, implementing a subset of the MIPS instruction set was simulated in C++. Structural hazards, data hazards, and control hazards were identified and were handled like in MIPS architecture. There were five pipelined stages and separate instruction cache and data cache to counter structural hazards in pipelined construction. Data hazards were countered by data forwarding except for load-use data hazards that needed an additional stall of one cycle. Its efficiency was compared to the implementation that stalls on every hazard. Cache and main memory were implemented in a hierarchy, and hence memory accesses like floating-point arithmetic operations caused variable delays and required stalling in the pipelined processor.

Course Project | Spring 2020 Prof. Kolin Paul

Arbitrary Precision Arithmetic

A library for basic arithmetic operations consisting of addition, subtraction, multiplication, division, absolute, power, and square root on arbitrary precision real numbers were written in C. This library was further used in functions created to handle arithmetic operations on arbitrary precision complex numbers, consisting of addition, subtraction, multiplication, division, and absolute. These functions were tested and compared on different inputs with pre implemented complex arithmetic functions in C. Program statistics were plotted using the GNU plot. A significant difference was observed in the performance of multiplication when implemented using the self-made library compared with the standard library, which was then optimized by using the Karatsuba algorithm.

Course Project | Fall 2019 Prof. Anshul Kumar

Text Encryption and Decryption

This project was written in VHDL and implemented on the BASYS-3 FPGA board using Xilinx ISE and Vivado. It had seven modules. The receiver module directed the FPGA board to receive input from gtkTerm installed on PC through USB serially. The input was 8-bit ASCII value of the character entered on gtkTerm, which was then passed to the private module where it was encrypted using a private key by stream cipher cryptography. It was then further passed to the memory module to store it at the address generated by the timing module. The timing module directed read and write in memory through read-enable and write-enable signals. The public module read the memory and encrypted the input further using a pseudo-random key generated using cantor pairing hash function with address and two other user-specified keys as parameters. The final encrypted input was then transmitted serially from FPGA to gtkTerm on PC by the transmitter module. The display module helped users interact with the FPGA board through switches, pushbuttons, LEDs, and seven-segment displays.

Database Management System

CData structures were created using an object-oriented programming language. LinkedList and HashTable were built to store data about students, hostels, courses, grades, and departments. Separate chaining using binary search trees and double hashing were two techniques used for collision resolution in HashTable, and their performances were compared on different input sets. An online selling Amazon-like platform was also simulated to check thread-safe operations on data structures.
